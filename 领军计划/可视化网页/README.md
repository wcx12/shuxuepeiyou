## 本地预览

建议从仓库根目录启动静态服务：

```powershell
python -m http.server 8080
```

然后访问：

```text
http://127.0.0.1:8080/领军计划/可视化网页/
```

也可以进入某个项目目录后单独启动 `python -m http.server` 进行预览。单独预览时，如果项目引用了上级目录资源，需要确认相对路径仍然有效。

## 维护约定

每个可视化项目单独放在一个目录中，目录命名使用 `YYYY-MM-DD-主题`。项目内保留 `index.html`、`styles.css`、`app.js`、必要的数据文件、必要的本地依赖和项目 README。

不要提交 `node_modules/`、本地检查截图、测试结果、临时构建产物或外部平台专用部署脚本。若页面必须依赖第三方库，优先说明来源和加载方式；课堂离线使用风险较高的依赖，建议放入项目本地 `vendor/` 目录。

新增可视化网页时，必须同步更新：

1. 本目录的 `index.html`。
2. 本目录的 `README.md`。
3. 新项目自己的 `README.md`。
4. 对应讲义目录中的 `可视化引用.md`。

如果暂时没有对应讲义，应在项目 README 中说明该网页的适用场景，避免误挂到不相关讲义。

## 发布检查

GitHub Pages 使用本仓库 `main` 分支根目录发布。更新可视化页面后，推送到 `main` 即可触发 Pages 更新。

提交前至少检查三项：入口页能够打开，项目页能够打开，浏览器控制台没有影响课堂演示的资源加载错误。
